# Personas — dieHantar Mobile

This document summarizes the primary user segments, their goals, motivations, and success criteria.

## 1) Passenger
- Description: End user requesting transport or food delivery.
- Goals: Fast booking, visible ETAs, low friction payments, price transparency.
- Motivations: Time savings, reliability, price/value.
- Success metrics: time-to-complete-booking ≤ 3 taps (for repeat orders), order completion rate, NPS.

## 2) Driver
- Description: Vehicle operator (motor/mobil) handling rides and/or deliveries.
- Goals: Optimize earnings, clear navigation, accurate job details, minimal idle time.
- Motivations: Earnings per hour, predictable acceptance rate, low carting time.
- Success metrics: trips/hour, acceptance rate, on-time arrivals.

## 3) Courier (Food Delivery)
- Description: Rider focused on collecting from restaurants and completing deliveries.
- Goals: Clear pickup windows, proof-of-delivery flow, consolidated route when possible.
- Success metrics: deliveries/hour, successful POD (photo) rate, distance per delivery.

## 4) Merchant / Restaurant
- Description: Business owner managing menu, orders, and preparation times.
- Goals: Accurate order details, push notifications for new orders, easy menu management.
- Success metrics: order acceptance time, decline rate, accurate ETAs.

## 5) Admin / Operations
- Description: Internal staff for incident handling, dispute resolution, and auditing.
- Goals: Fast access to logs, status dashboards, ability to intervene in trips/orders.
- Success metrics: mean time to resolve incidents, dashboard accuracy.

---

Each persona will feed into user journeys and acceptance criteria for the initial sprints.
